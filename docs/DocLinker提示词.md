第一阶段：项目初始化与系统级交互
目标：搭建 Tauri 2.0 基础环境，实现 Win11 亚克力窗口效果和全局热键。

Claude Code 提示词：
请初始化一个名为 "DocLinker" 的 Tauri 2.0 项目，前端使用 React (Vite) 和 Tailwind CSS。

具体窗口要求如下：

修改 tauri.conf.json：窗口需设置为无边框（frameless）、屏幕居中、且支持透明。

在 Rust 端实现全局快捷键 Alt + Space：用于切换（显示/隐藏）工具窗口。

实现“失焦隐藏”：当窗口失去焦点（Blur）时，自动隐藏到后台。

集成 window-vibrancy 库：为窗口应用 Windows 11 特有的“亚克力（Acrylic）”或“云母（Mica）”毛玻璃效果。

前端使用 Tailwind CSS 创建一个极简的搜索框界面。


第二阶段：工作空间管理与文件索引引擎
目标：实现 Rust 后端目录扫描、文件监听和 SQLite 数据库存储。

Claude Code 提示词：
现在，请在 Rust 后端实现核心索引引擎：

工作空间配置：创建一个基于 JSON 的配置系统，用于存储用户定义的目录路径（工作空间），并支持标签分类（Project, Doc, Scripts）。

SQLite 集成：使用 rusqlite 设置 SQLite 数据库。创建一个文件表（files）和一个 FTS5 虚拟表，用于对 .md 文件的全文内容进行索引。

文件扫描器：使用 walkdir 扫描配置的工作空间，提取 .md、.exe 和 .url 文件及其元数据并存入数据库。

实时监听：集成 notify 库监听工作空间的文件变化（创建/修改/重命名/删除），并实现数据库的增量更新。

Tauri 命令：为前端提供 API，用于添加/列出工作空间，并根据关键词对数据库进行模糊搜索。


第三阶段：搜索逻辑与 Markdown 预览界面
目标：前端实现搜索结果展示，并集成高性能的 Markdown 渲染。

Claude Code 提示词：
请完善前端界面和搜索逻辑：

搜索逻辑：当用户输入时调用后端搜索命令。排序权重为：别名 > 文件名 > 正文内容。结果列表需显示类别图标。

预览面板：实现分栏布局。当选中一个 .md 文件时，在右侧展示预览面板。

Markdown 渲染：使用 react-markdown 渲染内容，并集成 shiki 或 prismjs 对代码块进行语法高亮。

键盘导航：支持使用上下方向键切换搜索结果，按 Enter 键打开预览或执行文件。


第四阶段：命令提取、变量注入与自动化执行
目标：实现工具最核心的“动作”功能，让文档里的代码跑起来。

Claude Code 提示词：
最后阶段：实现针对代码块的“动作（Action）”系统：

代码块增强：在 Markdown 预览区，为每个代码块添加“复制”和“运行（播放图标）”按钮。

命令执行：创建一个名为 execute_command 的 Tauri 命令，用于调用 PowerShell 执行选中的代码块内容。

变量注入：使用正则识别代码块中的 {变量名} 占位符。如果存在变量，点击运行时需弹出 UI 输入框让用户填写，替换后再执行。

快速提取：实现快捷键 Ctrl + C。如果当前焦点在搜索列表的 Markdown 文件上，直接提取其第一个代码块到剪贴板。

安全机制：对于包含危险指令（如 rm, del, format）的代码，执行前必须弹出二次确认对话框。
